## Project Workspace Rules

Treat this folder as the implementation workspace for this project.

- Keep code, HTML, CSS, JavaScript, app scaffolding, assets, dependencies, generated files, build outputs, logs, and repos in this folder.
- Use the Obsidian vault at `C:\Users\cjcha\Documents\Obsidian\obsidian-ai-notes` as the documentation and project control layer.
- Keep project notes, goals, decisions, status, next actions, setup details, caveats, links, and handoff documentation in Obsidian.
- Check whether a matching project note already exists under `Projects/` in the Obsidian vault.
- If the project note is missing, create one from `Templates/Project Control Note.md`.
- Prefer a main note path like `Projects/<Project Name>/<Project Name>.md`.
- If the project grows, prefer a main overview note plus focused subpages instead of one very long page.
- Link the Obsidian note to this local folder and other durable resources such as repos, live sites, servers, and dashboards.
- Do not move the full code project into the Obsidian vault unless explicitly requested.
- Preserve unrelated vault changes and do not commit or sync broad vault edits without checking scope.
